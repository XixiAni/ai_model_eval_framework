AI大模型接口自动化评测框架（YAML数据驱动版）README
 
已同步日志模块 + 三重文件架构 + YAML数据驱动 + 批量评测执行功能
 
1 工具简介
 
封装基于requests.Session的AI大模型通用请求工具，统一处理请求头、超时、SSL校验、分层异常捕获、响应JSON标准化解析；内置完整日志模块，控制台+本地日志文件双输出，自动记录请求地址、入参、接口耗时、异常详情，输出标准化返回结构，完美适配AI自动化测试场景，支持无真实密钥的模拟返回调试。
新增拓展：搭载YAML数据驱动测试与批量自动评测能力，实现用例与代码彻底解耦；支持批量执行AI接口评测用例、自动统计通过率/耗时、生成执行汇总结果，成为完整可上线的AI测开简历项目。
额外优化：修复模型回复重复输出问题，区分控制台调试打印与轻量化持久化日志，避免日志文件存储大段对话文本。

3 完整代码逻辑说明
 
3.1 全局日志初始化模块
 
代码头部提供 init_api_logger() 日志初始化函数，生成专属日志实例：
 
1. 日志输出双通道：控制台实时打印 + 本地持久化文件 api_test.log；
2. 日志分级：INFO（正常请求/用例成功）、ERROR（请求异常/用例失败）；
3. 日志格式统一携带时间戳、日志等级、业务详情；
4. 做重复处理器防重逻辑，多次实例化工具类不会重复打印日志。
全局单例日志对象 global_logger，全文件复用。
 
3.2 库导入与环境预处理
 
导入requests网络库、json解析库、urljoin 处理接口地址拼接、全套requests网络异常类；新增 logging 日志库、time 计时库；过滤关闭SSL校验产生的警告，避免无关日志污染自动化测试输出。
main主脚本额外引入 python-dotenv 实现本地.env密钥文件自动加载，根据脚本绝对路径定位.env文件，不受运行终端目录影响。
 
3.3 类初始化 init 配置项
 
实例化时可传入鉴权密钥、基准域名base_url、自定义超时时间（默认30s），内部初始化如下属性：
 
- api_key：模型接口鉴权凭证
- base_url：接口域名前缀，用于和接口路径拼接完整请求地址，默认空字符串
- base_headers：全局请求头，预设JSON格式标识、Bearer鉴权头，自动绑定至Session
- timeout：全局请求超时时间
- verify：SSL证书校验开关，测试环境默认关闭
- session：持久会话对象，复用连接池，统一承载全局配置
- 新增 self.logger：绑定全局单例日志实例，类内所有请求方法复用该日志对象打印记录
密钥读取优先级：实例化传入api_key参数 > 系统环境变量 > 项目根目录.env文件变量；无有效密钥直接抛出ValueError并记录错误日志。
 
3.4 私有响应解析方法 _parse_response_json(resp)
 
接收请求返回的响应对象，标准化解析接口返回内容：
 
- 成功解析JSON：返回 code=0，data存储解析后的JSON字典，msg标记请求成功
- 解析失败（非JSON文本）：返回 code=-2，data存储接口原始文本，msg提示JSON格式错误
统一输出 code/data/msg 三层结构，方便自动化脚本做结果断言。
本方法无日志打印逻辑，仅做数据解析。
 
3.5 POST请求方法 send_post(api_path, request_data)
 
- 入参：api_path为接口路径（base_url为空时可直接传入完整URL）；request_data为请求体字典
- 拼接逻辑：内部使用 urljoin 拼接base_url与api_path，自动处理首尾斜杠，规避地址格式错误
- 新增请求前置逻辑：记录请求发起时间戳，打印INFO日志，记录完整请求地址与入参；
- 请求逻辑：调用session.post，通过json参数传入request_data字典（自动序列化到HTTP请求体），复用全局超时、SSL配置；使用raise_for_status自动识别非2xx错误状态码并抛出异常
- 正常响应：调用私有解析方法处理返回结果，计算耗时（毫秒），打印INFO日志记录请求耗时、HTTP状态码、业务返回code；
- 异常捕获：分层捕获超时、连接失败、HTTP状态码错误、其他网络异常，全部返回 code=-1 标记网络类错误；捕获异常后计算耗时，打印ERROR日志，完整记录错误原因、原始响应文本。
- 保留注释模拟返回代码块：注释打开后可跳过真实网络请求，纯测试框架调用逻辑。
 
3.6 GET请求方法 send_get(api_path, params=None)
 
- 入参：api_path使用规则同POST；params为URL查询参数字典（GET无请求体，参数会自动拼接在URL末尾生成 ?key=value 查询串）
- 新增请求前置逻辑：记录请求发起时间戳，打印INFO日志，记录完整请求地址与查询参数；
- 地址拼接、全局配置复用、状态码校验、异常捕获逻辑与POST完全一致
- 正常响应同样调用统一JSON解析方法，计算耗时并打印INFO日志、4xx/5xx状态码等。
 
3.7 自定义三层业务返回码说明
 
1. code=0：请求全流程正常（网络连通、HTTP状态码2xx、返回合法JSON）；
2. code=-1：网络/HTTP链路故障（超时、连接失败、4xx/5xx状态码等，未获取合法2xx响应）；
3. code=-2：HTTP通信正常，但返回内容无法解析为JSON，属于业务返回格式异常。
 
3.8 data字段作用
 
统一存放原始数据，方便测试脚本打印日志、定位问题：
 
1. code=0：存储接口完整返回JSON，可直接提取模型回答、token消耗等业务字段；
2. code=-2：存储接口原始文本，用于排查非JSON格式报错；
3. code=-1：赋值None，链路层面完全没拿到响应内容，无原始数据可存储。
 
3.9 日志模块设计补充
 
1. 无侵入改造：仅新增日志导入、初始化、打印代码，原有请求逻辑、返回结构、注释、模拟调试代码完全未改动；
2. 输出分层优化（重点更新）：区分控制台实时调试输出、持久化文件日志：
 
- 控制台print：完整打印模型回复文本，方便本地肉眼调试；
- 文件日志：成功用例仅记录简短标记 【测试用例执行成功】，不再存储大段模型对话，大幅缩减日志体积、消除重复输出；
- 失败用例日志完整留存错误信息、原始响应，保障故障排查能力；
 
3. 全局单例日志：所有AiApiRequest实例共用同一个日志对象，不会生成多个日志文件、不会重复输出日志。
 
3.10 主函数示例拓展说明
 
脚本底部 if name == "main" 自测示例完成输出逻辑优化：
 
1. 接口调用成功：控制台print完整模型返回内容，日志仅打印精简标记 【测试用例执行成功】，解决控制台+日志重复输出模型文本的问题；
2. 接口调用失败：控制台打印错误消息与原始数据，日志完整记录ERROR级别报错、错误消息、原始响应数据；
兼顾控制台即时打印 + 文件轻量化持久留存，适配批量自动化测试长期运行场景。
 
4 安全优化：环境变量读取API密钥
 
4.1 功能说明
 
支持从操作系统环境变量读取接口鉴权密钥，避免代码硬编码密钥，防止密钥提交代码仓库泄露，适配自动化测试流水线运行。
 
4.2 使用双模式
 
python
  
# 1. 手动传入密钥（本地临时调试）
client = AiApiRequest(api_key="sk-xxx", base_url="https://api.deepseek.com")

# 2. 自动读取环境变量（推荐，流水线/长期运行）
# 约定环境变量名称：AI_API_KEY
client = AiApiRequest(base_url="https://api.deepseek.com")
 
 
4.3 Windows系统临时配置环境变量
 
cmd
  
# CMD命令行
set AI_API_KEY=sk-xxxxxxxxxxxxxxxxxxx
 
 
powershell
  
# PowerShell
$env:AI_API_KEY="sk-xxxxxxxxxxxxxxxxxxx"
 
 
配置完成后直接运行脚本即可自动读取密钥。
 
4.4 容错机制
 
既未传入api_key参数，又不存在AI_API_KEY环境变量、.env文件未配置密钥时，工具类会主动抛出ValueError异常并记录ERROR日志，阻止无效请求发起。报错信息会同时给出三种解决方案：手动传参、系统环境变量、本地.env文件。
 
5 .env本地密钥配置方案（本地开发推荐）
 
5.1 作用
 
本地开发无需每次打开终端设置临时环境变量，项目根目录创建 .env 文件存放密钥，程序启动自动加载文件内变量，兼顾便捷性与安全性。
 
5.2 使用步骤
 
1. 在项目根目录新建文件，文件名固定为 .env

*.pyo
*.pyd
# 虚拟环境文件夹
venv/

4. 新建 .env.example 模板文件提交仓库，给其他开发者参考：

5.3 加载逻辑说明
 
main主脚本通过文件绝对路径定位.env文件并加载，不受执行脚本的终端工作目录限制，任意目录运行均可正常读取密钥文件。
 
5.4 优先级说明
 
环境变量生效优先级：系统全局环境变量 > 终端临时环境变量 > .env文件变量；本地开发优先使用.env，线上服务器部署使用系统全局环境变量。
 
6 新增核心功能：YAML数据驱动批量评测（AI测开项目一）
 
6.1 三大核心文件分工（三重架构）
 
1. api_client_decoupling.py：底层请求核心
 
- AiApiRequest：AI接口请求工具类，负责发请求、日志、异常处理、响应标准化
- YamlCaseLoader：YAML用例加载器，负责读取/校验评测用例，过滤非法用例
 
2. ModelEvalRunner.py：业务执行核心
 
- ModelEvalRunner：批量评测执行器，负责循环执行用例、收集结果、异常容错
 
3. main.py：项目入口核心
 
- 加载环境变量、初始化客户端、加载YAML用例、触发批量执行、输出汇总结果
 
6.2 YamlCaseLoader（YAML用例加载器）
 
- 职责：读取本地YAML用例文件、校验必填字段（case_id/case_desc/api_path/request_body）、过滤非法用例
- 特性：完全解耦，不依赖请求工具，独立负责用例解析
- 输出：标准化合法用例列表，供给执行器使用
 
6.3 ModelEvalRunner（批量评测执行器）
 
- 职责：接收YAML用例、循环执行单条接口请求、捕获执行异常、收集全量结果
- 特性：单条用例失败不中断整批任务，记录耗时/时间戳/成功标记/错误信息
- 输出：完整评测结果列表，用于统计、导出、断言
 
6.4 完整运行流程
 
1. main.py加载.env密钥 → 初始化AiApiRequest客户端
2. YamlCaseLoader读取chat_cases_new.yaml并校验用例
3. ModelEvalRunner注入客户端，批量执行所有用例
4. 自动统计总用例数、成功数、失败数、通过率
5. 控制台输出可视化汇总结果，日志全程记录执行过程
 
6.5 批量评测核心优势
 
1. 数据驱动（DDT）：用例写在YAML，无需改Python代码
2. 零侵入底层：不修改AiApiRequest任何原有逻辑
3. 高容错：单条用例异常不影响整体执行
4. 可视化结果：自动计算通过率，适合演示与简历展示
5. 易拓展：结果可直接对接CSV导出、响应断言、测试报告生成

9 拓展功能模块（新增）
 
9.1 批量评测统计（ResultStatistics）
 
- 功能：基于批量执行结果自动计算总用例数、成功数、失败数、通过率、总耗时、平均耗时、最大/最小耗时
- 输入：ModelEvalRunner输出的完整结果列表
- 输出：控制台可视化统计报告，支持后续报表生成
 
9.2 CSV评测结果导出（CsvResultExporter）
 
- 功能：自动创建report目录，将所有用例执行记录导出为CSV文件
- 导出字段：用例ID、用例描述、执行时间、请求耗时、执行状态、错误信息、断言结果
- 兼容性：使用UTF-8-SIG编码，Excel直接打开无乱码，可用于测试报告归档
 
9.3 通用响应断言工具（ResponseAssertor）
 
- 功能：读取YAML用例中expect字段，自动完成响应结果校验
- 支持断言类型：接口返回code校验、返回内容关键词包含校验
- 特性：断言结果自动存入执行报告，失败自动输出错误信息，无需手动编写判断逻辑

10. 新增安全与稳定性优化
 
10.1 日志密钥脱敏优化
 
对接口鉴权请求头进行脱敏处理，采用敏感信息打码机制（同手机号脱敏逻辑），保留前缀与后缀特征，中间隐藏核心密钥内容，杜绝调试/日志输出时密钥明文泄露风险，满足企业级安全开发规范。
 
10.2 网络异常自动重试机制
 
针对请求超时、瞬时网络连接失败场景增加自动重试能力，支持配置最大重试次数、重试间隔，网络抖动时自动重试1次，有效避免因网络波动导致用例误失败，大幅提升批量评测执行稳定性。
